Note is the system's in-flow callout — an alternative to Toast when the signal belongs inline with the content rather than in the corner of the viewport. Two decisions stand out:

- **Outline-only default.** The filled tint is opt-in via `fill`, so Notes read as quiet annotations until explicitly elevated. On a dense form, a row of filled Notes feels shouty; outline-only sits within the layout rhythm.
- **`alert` vs `warning` duplication.** Both use the same amber tokens. The split is semantic: `warning` is a stateful validation signal, `alert` is a transient urgency signal. They're intentionally indistinguishable visually so the UI doesn't leak an internal taxonomy to the user.

The `label` prop is unusual — passing `true` auto-capitalises `type` as the prefix, which keeps short callouts single-line without forcing the author to restate the semantic name.
